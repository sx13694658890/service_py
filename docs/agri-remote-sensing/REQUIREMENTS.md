# 农业遥感应用 · 需求说明

本文档描述面向农业生产管理的遥感监测类产品的**需求范围与验收口径**，供产品、设计、前后端与算法/数据协同评审。  
若与后续高保真设计或监管要求冲突，以评审结论为准。

---

## 1. 背景与目标

### 1.1 背景

- 作物长势、旱涝、病虫害风险等与**时空连续的植被与地表状态**强相关，遥感（卫星、航空、无人机）可提供大范围、可重复的观测手段。
- 目标用户可能包括：种植企业/合作社农技人员、区域农业服务人员、农业保险与金融风控辅助场景（本期以**监测与决策辅助**为主）。

### 1.2 目标（MVP）

- 在 Web 端完成「**选地 → 选时相 → 看指数/图层 → 简单解读与导出**」闭环。
- 支持至少一种**植被指数**（如 NDVI）的时序展示与当前时相空间分布查看。
- 数据与权限边界清晰：**谁可以看哪块田、哪段历史**可追溯。

### 1.3 成功标准

- 用户在 **5 分钟内**完成：登录 → 进入地块/区域 → 查看最近一期指数概况 → 打开时序曲线。
- 核心页面在 **1080p** 宽度下主要操作无需横向滚动；地图与图表在弱网下**有加载与失败提示**，不出现白屏。
- 对外演示可说明**数据来源、更新频率与精度限制**（见 §8）。

## 2. 用户与场景

### 2.1 角色（可迭代）

| 角色       | 诉求概要 |
|------------|----------|
| 农技用户   | 查看自己负责地块的长势与异常时段 |
| 管理员     | 维护组织、地块、成员与数据订阅范围 |
| 只读访客   | 演示或上级抽查（可选，MVP 可合并为普通用户） |

### 2.2 典型场景

1. **长势巡检**：按地块列表进入，查看最新 NDVI 分级图与相对上周/上月变化。
2. **时序对比**：选择时间范围，查看指数曲线，标注农事事件（施肥、灌溉，可选）。
3. **异常初筛**：指数低于阈值或环比骤降时列表或地图上高亮（MVP 可做简单阈值规则）。

## 3. 功能需求

### 3.1 组织与权限（MVP 可简化）

- 用户属于组织；地块（或「监测单元」）归属组织。
- **数据隔离**：仅可访问本组织授权范围内的地块与遥感产品。
- 登录、登出与 401 处理与现有工程一致。

### 3.2 地块与空间范围

- 地块以**矢量边界**表示（GeoJSON 或多边形 WKT）；支持在地图上**缩略展示**与**点击查看属性**（名称、面积、作物类型可选）。
- 支持按名称搜索地块（MVP）；空间绘制编辑可放到二期。

### 3.3 遥感产品

- **指数**：MVP 至少 **NDVI**；二期可扩展 EVI、NDWI（水体/墒情相关）等。
- **时间**：每期产品带**观测日期**（或合成周期起止）；列表或时间轴选择。
- **空间**：以**瓦片或 COG 切片服务**形式在地图上调阅；或与后端返回的**统计值**（地块均值）驱动图表。

### 3.4 地图与可视化

- 底图：影像/矢量底图可配置（注意合规与密钥管理）。
- 图层：指数分级着色、透明度调节、图例说明。
- 图表：单地块指数**时序折线图**；支持选择日期范围（MVP 可限制最长区间，如 1 年）。

### 3.5 导出与分享（可选 MVP）

- 导出当前视图为图片或 PDF（二期优先）。
- 分享只读链接（二期，需鉴权与过期策略）。

### 3.6 非功能需求

- **性能**：首屏可交互时间目标（与现有前端指标对齐，如 3s 内骨架+地图容器就绪）。
- **可用性**：瓦片/接口失败时**错误提示与重试**；指数缺失时**说明文案**（如云量过大无有效值）。
- **可访问性**：关键按钮与图层开关有可读标签。
- **合规**：遵守测绘与遥感数据使用规定；不在前端暴露长期有效的第三方密钥。

## 4. 数据与接口（需求级）

以下由后端/数据团队细化 OpenAPI，前端按契约对接。

- `GET` 组织下地列表、地块详情（含几何）。
- `GET` 地块在某指数、时间范围内的**时序序列**（日期 + 数值 + 可选质量标记）。
- `GET` 某期产品的**瓦片模板 URL** 或 **COG 访问地址**（带短期签名或网关鉴权）。
- 可选：`POST` 上传/修正地块边界（二期）。

### 4.1 与本仓库服务端现状对照（补充）

便于评审「需求 vs 已实现」，下列为 **server_python** 当前能力与缺口（随迭代更新）。

| 需求方向 | 现状 | 建议服务端补充 |
|----------|------|----------------|
| 动态 COG 瓦片（TiTiler） | **独立 Docker/进程** 部署官方 TiTiler；**本仓库不包含内嵌 TiTiler**。见 [TITILER_STANDALONE.md](./TITILER_STANDALONE.md)。 | 网关、监控、超时；COG 仅 TiTiler 可达时的网络策略。 |
| 业务层：产品元数据 + 安全瓦片 URL | **未实现** `/api/v1/agri/...`。 | 提供 `GET .../products/{id}/layer`（或等价）返回 **已鉴权场景下** 的 `tileJsonUrl` / `urlTemplate`，其中 **COG 地址为对象存储短期签名 URL**，避免前端拼接裸 `url=`。 |
| 地块矢量、组织权限、时序统计 | **未实现** PostGIS 与农业业务 API。 | 按 §4 列表实现 `parcels`、`timeseries`；与 TiTiler 仅通过「后端下发的 COG URL」衔接。 |
| 瓦片请求鉴权 | TiTiler 为**独立服务**，与本仓库 JWT 无直接耦合。 | 网关鉴权、内网部署、API Key、或仅预签名 COG URL；限制 `url` 来源以防 SSRF/盗链。 |
| 跨域 | 本仓库业务 API 默认未全局开启 CORS。 | 前端直连 **TiTiler 域名** 拉瓦片时，在 **TiTiler 侧或网关** 配置 CORS，或**同源反代**（见 [TITILER_STANDALONE.md](./TITILER_STANDALONE.md)）。 |

前端对接、MapLibre 示例与参数见 [TITILER_FRONTEND.md](./TITILER_FRONTEND.md)；**独立部署方式**见 [TITILER_STANDALONE.md](./TITILER_STANDALONE.md)。

## 5. 范围边界

### 5.1 本期（MVP）包含

- Web 端地图 + 单指数（NDVI）+ 时序曲线 + 基础权限与地块列表。
- 与现有认证体系打通。

### 5.2 本期不包含（可入路线图）

- 像素级病虫害识别、产量预测模型产品化。
- 无人机航片全流程处理管线（拍摄计划、拼接、空三）自建。
- 移动端原生 App（可用响应式 H5 代替）。

---

## 6. 验收清单（MVP）

- [ ] 登录后可看到授权范围内的地块列表并可定位到地图。
- [ ] 可选择至少一期 NDVI 产品在地图上叠加显示，图例可读。
- [ ] 至少一个地块可展示 NDVI 时序曲线，日期范围可选。
- [ ] 云量高/无数据时有明确提示，不崩溃。
- [ ] 接口失败可重试或返回错误态，无空白页。

---

## 7. 依赖与风险

- **数据源**：Sentinel-2、商业卫星或第三方 API 的配额、延迟与覆盖度。
- **几何精度**：地块边界与像元配准误差需在文档中声明。
- **成本**：瓦片流量与对象存储费用随用户量上升。

---

## 8. 术语与说明（面向演示）

- **NDVI**：归一化植被指数，常用于表征绿色植被活力；受土壤背景、大气、云影响，需结合质量掩膜与农艺知识解读。
- **合成产品**：多日合成可降低云影响，但日期代表「窗口内」而非单日快照。
