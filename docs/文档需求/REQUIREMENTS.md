# 文档中心页面需求文档（基于截图）

本文档根据当前页面截图提炼需求，用于产品、设计、前端、后端协同评审。  
若与后续高保真设计稿冲突，以设计稿和评审结论为准。

---

## 1. 背景与目标

### 1.1 背景

- 当前已有文档中心首页雏形：左侧导航、顶部工具区、中间文档卡片列表。
- 需要将现有界面固化为可开发、可验收的产品需求，减少“看图开发”的歧义。

### 1.2 目标

- 明确页面信息架构（导航、列表、操作、状态）。
- 支持用户快速查看文档、进入专题、按角色访问常用入口。
- 保留通知/刷新/个人入口，形成统一工作台体验。

### 1.3 成功标准

- 用户 3 步内完成“进入页面 -> 看到文档列表 -> 点击阅读”。
- 左侧导航分组清晰，首屏核心内容可直接访问。
- 页面在 1366 宽度下无需横向滚动，关键操作可见。

---

## 2. 设计输入（截图观察）

从截图可识别的页面元素：

- 顶部：通知图标、刷新图标、辅助图标（如工具/设置）、用户头像区（`ME` + 在线点）。
- 左侧导航：多分组结构，含「首页」「使用文档」及业务菜单项。
- 中间区域：
  - 欢迎 Banner（“欢迎查阅文档”）。
  - 「全部文档」区块。
  - 多条文档卡片：标题、摘要、标签/按钮（如“预览”）、右侧评分/标识。
- 左下角有浮动入口（疑似客服/快捷入口）。

---

## 3. 功能需求

### 3.1 页面结构

- 页面分为 `Header + Sidebar + Content` 三大区域。
- `Sidebar` 固定在左侧，`Content` 支持纵向滚动。
- `Header` 固定在顶部，展示全局快捷操作和账号入口。

### 3.2 顶部区域（Header）

- 展示以下操作入口（从右向左）：
  - 通知入口（可接入站内消息未读态）。
  - 刷新入口（刷新当前文档列表数据）。
  - 预留工具入口（与产品后续能力联动）。
  - 用户区（名称/头像、在线状态）。
- 用户区点击后弹出菜单（最少包含：个人中心、退出登录）。

### 3.3 左侧导航（Sidebar）

- 按业务分组展示菜单，分组标题示例：`回访`、`财务`、`用户`。
- 菜单项支持：
  - 默认态 / hover / active 态；
  - 二级高亮（当前路由对应菜单高亮）；
  - 可选 badge（如 `New`）。
- 顶部应有品牌区（Logo + 折叠按钮预留）。

### 3.4 文档主页内容（Content）

- 顶部欢迎 Banner：
  - 显示欢迎文案与引导描述；
  - 预留插画位（截图已有喇叭图形）。
- 文档列表区：
  - 区块标题：`全部文档`。
  - 卡片字段：`标题`、`摘要`、`操作按钮（预览）`、`评分/标签展示区`。
  - 卡片按创建时间或推荐权重排序（后端可配置，默认按最新）。

### 3.5 文档卡片交互

- 点击卡片标题或“预览”进入文档详情页。
- 鼠标悬停卡片有弱高亮/阴影反馈。
- 对于无权限文档，显示禁用态并给出提示。

### 3.6 浮动入口（左下）

- 保留浮动按钮位（可用于客服/快捷反馈）。
- 不与主要导航产生遮挡；低分辨率下允许自动收边。

## 4. 数据需求

### 4.1 文档列表最小字段

- `id`：文档 ID
- `title`：标题
- `summary`：摘要
- `category`：分类（可选）
- `score`：评分（可选）
- `tags`：标签数组（可选）
- `created_at`：创建时间
- `updated_at`：更新时间
- `can_view`：是否可查看

### 4.2 列表查询建议

- 支持分页：`limit`、`offset`
- 支持筛选：分类 / 关键词（后续迭代）
- 支持排序：`latest`、`popular`（后续迭代）

---

## 5. 非功能需求

- 性能：首屏可见内容在正常网络下应快速渲染（建议首屏骨架屏）。
- 可用性：接口失败时显示可重试提示，不出现空白页。
- 可访问性：关键按钮可键盘访问，图标有可读文本提示。
- 响应式：1366 为主，兼容 1200+；更小宽度可降级为折叠菜单。

---

## 6. 验收标准（MVP）

- 进入页面后可见完整三栏结构（顶部、左侧、中部）。
- 左侧菜单可切换并高亮当前项。
- 中部至少展示 5 条文档卡片，支持点击“预览”。
- 顶部通知/刷新/用户菜单可交互（通知内容可先占位）。
- 异常态（无数据/接口失败）有可见提示。

---

## 7. 范围边界

### 7.1 本期包含

- 文档首页信息架构、列表展示、基础交互。
- 与现有登录态打通（用户区显示当前账号）。

### 7.2 本期不包含

- 富文本编辑器、文档在线编辑协同。
- 复杂权限管理后台（仅消费现有权限结果）。
- 多语言切换与主题系统（可后续评估）。

---

## 8. 依赖与风险

- 依赖后端提供稳定文档列表接口与权限字段。
- 若评分/标签来源不稳定，前端需支持“字段缺失兜底展示”。
- 菜单项较多时需评估信息架构，避免二级导航混乱。
