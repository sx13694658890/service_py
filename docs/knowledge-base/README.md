# AI 助手知识库（维护说明）

本目录为 **首页 AI 问答** 的检索知识来源：服务启动时加载全部 `.md` 切块建 BM25 索引，并与根目录下 **`quick_questions.yaml`** 配合（快捷问题的 `retrieval_query` 会参与检索加权）。

## 目录约定

| 文件 | 用途 |
|------|------|
| `quick_questions.yaml` | 快捷问题 id / 文案 / 检索提示 / 转人工路由 |
| `00-*.md` ~ `05-*.md` | 按主题拆分的产品说明与 FAQ（**用 `## 二级标题` 分节**，便于切块） |
| `README.md` | 本说明（**不参与检索**：无 `## ` 业务节时可整块较短，或保持为说明文） |

新增主题时：优先 **新建一个编号 md** 或 **在对应文件增加 `## 小节`**，避免单文件过长（建议每节控制在数千字内，超长会被自动再切）。

## 修改后如何生效

- 开发：重启 API 进程；或代码中调用 `reload_knowledge_bundle()` 后按需自行接入热更新。
- 生产：部署重启后重新加载。

## 与需求文档的关系

内容对齐 `docs/ai问答需求/REQUIREMENTS.md` 中的产品范围、交互与验收要点；若产品变更，请 **同步改本目录正文**，并视需要调整 `quick_questions.yaml` 的 `retrieval_query` 关键词。

## 环境变量

可通过 `AI_KNOWLEDGE_DIR` 指向其他目录（绝对路径或相对路径）；不设则默认使用本目录（见服务端 `default_knowledge_dir`）。
